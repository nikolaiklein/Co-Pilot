# Отчет о проекте "Telegram-бот Ко-пилот (Биограф/Коуч)"

## Обзор проекта

Данный проект представляет собой интеллектуального Telegram-бота, разработанного на Python с использованием FastAPI. Бот выполняет роль эмпатичного ассистента-биографа и коуча, специально настроенного для работы с мужчиной 1970 года рождения. Его цель — интервьюировать пользователя, выявлять навыки, структурировать опыт и помогать в освоении современных ИИ-инструментов.

Проект спроектирован для развертывания в Google Cloud Run (Serverless) и использует современные облачные технологии Google (Firebase, Gemini).

## Стек технологий

*   **Язык программирования:** Python 3.11
*   **Web-фреймворк:** FastAPI (асинхронный, высокопроизводительный)
*   **Сервер приложений:** Uvicorn
*   **Telegram Bot API:** `python-telegram-bot` (v20+, асинхронная версия)
*   **База данных:** Google Cloud Firestore (NoSQL, документо-ориентированная)
*   **Искусственный интеллект (LLM):** Google Gemini 1.5 Pro (через SDK `google-genai`)
*   **Аутентификация:** Google Application Default Credentials (ADC)
*   **Контейнеризация:** Docker (образ `python:3.11-slim`)

## Подробное описание компонентов

### 1. `main.py` (Точка входа)
*   Инициализирует FastAPI приложение.
*   Настраивает жизненный цикл приложения (`startup`, `shutdown`), инициализируя сервисы (DB, AI, Bot, Analyzer).
*   Предоставляет эндпоинты:
    *   `GET /`: Health check (для проверки работоспособности Cloud Run).
    *   `POST /webhook`: Принимает обновления от Telegram и передает их в `bot_app`.
    *   `POST /cron/analyze`: Запускает фоновый процесс анализа профиля пользователя (для Cloud Scheduler).

### 2. `services/db.py` (База данных)
*   Класс `DatabaseService` инкапсулирует работу с Firestore.
*   **Основные методы:**
    *   `get_or_create_user`: Создает профиль пользователя при первом обращении.
    *   `save_message`: Сохраняет историю переписки (роли `user` и `assistant`) с временными метками сервера.
    *   `get_last_messages`: Извлекает контекст диалога (последние N сообщений) для отправки в LLM.
    *   `update_user` / `save_report`: Сохраняет результаты работы аналитического ядра.

### 3. `services/ai_engine.py` (Интеллект)
*   Класс `AIEngine` использует `google-genai` SDK для общения с Gemini 1.5 Pro.
*   **Системный промпт:** Задает роль "эмпатичного биографа и коуча", стиль общения и задачи.
*   **Функции:**
    *   `generate_response`: Генерирует текстовый ответ на основе истории диалога.
    *   `transcribe_audio`: Использует мультимодальные возможности Gemini для транскрибации голосовых сообщений (STT) напрямую из аудиобайтов (OGG/Opus).
    *   `analyze_content`: Выполняет специфические задачи анализа (без контекста чата) по запросу.

### 4. `services/telegram_bot.py` (Бот)
*   Настраивает `Application` из `python-telegram-bot`.
*   **Обработка сообщений:**
    *   **Текст:** Сохраняет сообщение -> Формирует контекст -> Запрашивает ответ у AI -> Сохраняет ответ -> Отправляет пользователю.
    *   **Голос:** Скачивает файл в память -> Транскрибирует через AI -> Обрабатывает как текст (с пометкой `[Голосовое сообщение]`).
*   Отправляет статусы "печатает" (`typing`) или "записывает голос" (`upload_voice`) для улучшения UX.

### 5. `services/analyzer.py` (Аналитическое ядро)
*   Класс `AnalyzerService` запускается отдельно от основного цикла чата (через cron).
*   Анализирует последние 50 сообщений пользователя.
*   Формирует структурированный JSON-отчет:
    *   Новые навыки.
    *   Интересы/Хобби.
    *   Проблемы/Боли.
    *   Мечты.
    *   Краткое резюме.
*   Обновляет профиль пользователя в БД, позволяя боту "лучше узнавать" собеседника со временем.

## Развертывание и настройка

### Необходимые переменные окружения (.env)
*   `TELEGRAM_BOT_TOKEN`: Токен бота от @BotFather.
*   `GEMINI_API_KEY` (или `GOOGLE_API_KEY`): Ключ API для Google AI Studio / Vertex AI.
*   `GOOGLE_APPLICATION_CREDENTIALS`: Путь к JSON-ключу сервисного аккаунта (только для локального запуска).
*   `PORT`: Порт сервера (по умолчанию 8080, автоматически задается в Cloud Run).

### Запуск локально
1.  Установить зависимости: `pip install -r requirements.txt`
2.  Настроить `.env`.
3.  Запустить сервер: `uvicorn main:app --reload` (или через Docker).

### Деплой в Cloud Run
1.  Собрать образ: `docker build -t gcr.io/PROJECT_ID/bot-image .`
2.  Запушить: `docker push gcr.io/PROJECT_ID/bot-image`
3.  Развернуть сервис, указав переменные окружения и привязав сервисный аккаунт с правами Firestore и Vertex AI User.
4.  Настроить Webhook Telegram на URL сервиса: `https://api.telegram.org/bot<TOKEN>/setWebhook?url=<CLOUD_RUN_URL>/webhook`
